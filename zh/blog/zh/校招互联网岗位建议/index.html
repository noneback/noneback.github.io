互联网岗位类型

技术岗

关键词：开发、算法

开发

方向大体上可以分为：前端、后端、测试。

前端

前端关注页面设计以及用户的交互体验。具体可以细分为web开发和客户端开发。

web开发简单来讲开发出美观好用的网页，但其实很多的页面都是网页的范畴。比如普通网页，微信小程序，桌面客户端（electron），移动客户端（Dart / React-Native）

客户端开发请参考其他同学的分享。

后端

主要分业务开发和基础架构两种。

业务开发、注重自身对业务的理解，把业务流程抽象成代码。就工作内容来说就是数据的增删改差，比如xxx管理系统，只是系统交互更加复杂。人员对接频繁，系统迭代快。

基础架构，看重计算机基础技术，一些经典的方向比如数据库、消息队列等基础软件开发。开发周期相对长一些，系统稳定性最重要，产品成熟后可能会沦为运维。

测试

测试岗有纯测试测试开发。纯测试主要就是在测试产品功能逻辑（点击或者写代码）。测试开发除了测试可能也会开发测试工具。

算法

算法岗一般是指机器学习和数据挖掘等算法。据我了解，一般本科生没有相关成果（论文等）或者实习经历，相对比较难找到这类的正式工作。我也没有相关经验，所以抄了一下知乎回答

职能岗

关键词：HR、运营、产品

东霖应该会分享相关内容。

如何准备

简历

简历一般需要通过HR和小组Leader的筛选才能进入面试阶段。

尽量控制在一页，不写与岗位无关的信息，突出自己的优势（学校、奖项、排名、技能、etc）

参考文章

  • 个人简介：学校 + 专业 + 相关课程 + 奖项 + 技能

  • 个人项目 + 实习经历：没有合适的可以写一下课程大作业之类的项目，主要目的是丰富简历以及引导面试官提问

  • 其他 : github / blog / 专业证书等一些能表现技能和优势的东西

投递

岗位选择

总的一般对本科生来说，技术岗面试的难度：测试 < 前端 < 后端 < 算法

难度由 投递人数 + 岗位数量 + 小组要求 共同决定，代表竞争压力，与岗位好坏弱相关

建议按自己的预期，有策略的投递。

投递渠道

熟悉的学长学姐内推 > 学长学姐内推 > 网友内推 > 自己投递

内推主要是让内推人能关注你的面试流程，及时给你反馈，有一些公司内推可以跳过一些流程（如笔试）加快进度。

内推的帖子建议关注北邮人论坛相关版块

面试

一般都是 两轮以上技术面 + 一轮HR面

面试准备

面试内容基本就是 计算机基础 + 简历相关内容 + 算法。准备的话，牛客网面经 + 复习简历相关内容 + LeetCode高频题。

面试流程

  • 自我介绍：不算太重要，大方的简单介绍一下就好
  • 面试官提问：重要，针对简历提问，不会可以直说，不要装懂
  • 做算法题：重要，手撕代码
  • 反问环节：不算太重要，可以问问工作内容之类的

建议

  • 尽早多投递：完全准备好是不现实的，建议准备差不多了就尽早投递。多投递一些公司，多积累面试经验，多在面试后总结。刚开始难免失败，面试多了其实面试内容都差不太多
  • 拿到心仪的offer之前，请保持干劲。
